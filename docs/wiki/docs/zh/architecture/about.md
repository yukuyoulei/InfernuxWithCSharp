# 为什么会有Infernux

> 和技术无关，讲一点小故事。做引擎的路上一直没遇到志同道合的、愿意一起研究这种资本市场都已经抛弃了的老古董的朋友，索性把一点这段时间的想法记在这里，希望十年之后回过头来看，能博我自己会心一笑吧。

第一次想要做一个游戏引擎还是在大三，那时候我比现在年轻一点。刚看完Games101、跟着HK-SHAO大佬的[Taichi: 从零开始的光线追踪 | HK-SHAO](https://shao.fun/blog/w/taichi-ray-tracing.html)手搓了一遍基于taichi的实时光追、复现了几个简单的后处理效果，就觉得自己好像已经用无形的大手紧紧拿捏住整个图形学的地基，可以开宗立派了。正好此时Unity宣布收取安装费的新闻铺天盖地，我就开始想：要是能用手上的taichi做渲染和并行计算，做一个完全由Python驱动的游戏引擎，我岂不是可以扬名立万了——到时候比肩一众大佬也不为过。

这个想法放弃于第二个月手搓光栅化器的时候，我发现我写出来的光栅化还没有光追高效。光是跑起来基本的面裁剪算法，就把整个引擎的帧率拉到了100fps上下波动，成为受人景仰的图形学大手子的梦顷刻惊醒。后来又因为保研的论文要求（我的本科是一所双非学校，如果去一个比较好的院校读研，论文几乎是必需品），做了一段时间的新式图形学，包括但不限于3DGS，AIGC等等。最焦虑的那段时间，正好GPT3.5横空出世，并开始被应用于解决大家发论文的需求这个领域上，我也开始转向大模型相关的方向去做一些工作。

> 那是一个所有论文都是Prompt工程的时代。印象最深的一篇A会论文，主要工作是让一个LLM对特定问题进行多次反复推理，然后以少数服从多数的原则来取正确答案。

引擎相关的事情也就暂时停滞了。

后来凭借贵人的赏识，和一些故事会级别的小工作以及不错的运气，我来到了一所不错的学校读研，继续研究大模型。大概是在24年的年末某一天，我在深圳城中村出租屋里半夜惊醒，看着美的空调发出了绿色小亮光和呜呜呜的上下扫风声，就像很多传说故事里那种龙场悟道的情节，倏忽想起了我的前半生——想起了高中编造这样那样的理由玩智能手机、翻墙去网吧等故事，想起了大模型和人类的未来，于是心生大欢喜，又有大感动，就好像有得道高僧在我耳朵里开示微言法义。只可惜那么多大义经文怎么都听不清、记不住，就只知道我不再想继续搞大模型了，做一点我真正喜欢的东西。

当然，不是说我与净土佛国有缘，而是说我想要立刻回头去做图形学：做大模型的日子始终少了点能具象化出来的成就感，`system_message="you are a helpful assistant"`让我逐渐迷失在了沉重的故事里。我并不否认大模型本身是一个很有潜力的方向，也是目前看来AGI实现的最近途径，但我就是提不起兴趣。于是第二天刚醒，我就开始找实习，看看有没有什么能让我重回CG的实习岗位。

后来我去了深圳的一家具身智能公司实习。公司自研的仿真引擎，底层是C++，上层用Python调用——和你现在看到的Infernux一模一样。

那几个月，我从同事和mentor那里学到了很多东西（感谢他们，真的教会了我很多东西，让我得以蜕变）。从怎么组织工程结构，到怎么踩坑、怎么填坑，再到怎么把一个实时光追渲染器从零搭起来。他们给我很大的自由度，我甚至能在很大程度上挑拣自己想修的issue，我也乐得多加加班。

实习结束的时候，我提了不少PR，也终于知道一个“能用的引擎”应该长什么样。更重要的是，我重新开启了那个大三写到一半就丢下的项目，并从头开始构建，让它真的变成一个合适的引擎。这个引擎的名字也是在那时候定下来的，Infernux这个名字也是在下班路上的地铁上和dpsk聊着聊着取出来的，大概是Inferno + lux的意思，拉丁文里的地狱火和光。既是自嘲，也是实话，那些通宵调 Vulkan 屏障的日子，那些物理碰撞死活对不齐的夜晚，确实像在地狱里走了一遭。但是星星之火，可以燎原嘛。指不定这团火能烧出点什么新的东西来。

于是一年过去了，你就看到了现在的Infernux。中间有一段时间它也被我简称为InfEngine，但是好巧不巧，arxiv上最近挂上了一篇论文，提出的方法也叫InfEngine，于是我只能在大规模推广的前期临时去掉这个简称，而只叫Infernux了。不过我不保证清理得完全干净，所以你也有可能在某些地方看到残留的InfEngine这个简称。

不管怎么说，Infernux的0.1版本终归是写出来了。它已经收获了我自己的一些喜欢，也希望它能收获一些别人的关注吧。
